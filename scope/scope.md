# Assessment Scope Definition

## Assessment Type
This engagement represents a simulated enterprise web application security assessment
focused on identifying realistic security risks rather than exhaustive vulnerability discovery.

## In-Scope Assets
The following components were included in the assessment:

- Customer-facing web application
- Authentication and session management mechanisms
- Authorization and role-based access control
- Backend API endpoints
- User and CSR role separation

## Out-of-Scope Assets
The following activities and components were explicitly excluded from this assessment:

- Denial of Service (DoS) attacks
- Social engineering attacks
- Physical security controls
- Network-level infrastructure testing

## Assumptions
- The application represents a production-like enterprise environment
- All testing was conducted in an authorized and controlled lab environment
- No real customer data was used during the assessment

## Limitations
- Source code review was not performed
- Mobile application components were not assessed
